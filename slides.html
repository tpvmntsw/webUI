怎麼做一個畫面、怎麼排 z-order、怎麼交付

架構：shell + screens

shell.js（我們維護）

  • 誰蓋誰（z-order）
  • 按鍵路由 NAV
  • WebSocket 跟後端
  • 開關畫面 / 逾時自動關

screens/*.js（你寫）

  • 一支檔 = 一個畫面
  • 音量條、OSD、Information…
  • 只填一個 Shell.register
  • 只碰自己的 DOM

新功能 ＝ 加一支 .js + index.html 一行。

兩種「疊」別搞混

疊的層次誰負責怎麼控制
畫面 vs 畫面
音量條蓋在 OSD 上？		shell你宣告的 layer 名
畫面內部物件
按鈕疊在縮圖上？		你自己普通 CSS z-index

畫面內部愛怎麼疊就怎麼疊，跳不出你的容器。
shell 只管畫面之間誰在上面 → 你只挑一個 layer 名。

你要做的事 3 步

# 1. 複製範本
cp screens/_template.js screens/audio-volume.js

# 2. 填一個 Shell.register({ ... })  （下一頁）

# 3. index.html 加一行
<script src="screens/audio-volume.js"></script>

重整瀏覽器 → 完成。沒有 build、沒有 npm、沒有框架。

一個畫面長這樣

Shell.register({
  id: 'audio-volume',        // == 檔名去 .js
  layer: 'volume',             // 層級「名字」，不是數字
  timeout: 15000,             // 15s 沒動作自動關

  mount: function(el){ /* 建 DOM 一次，只碰自己的 el */ },
  onShow: function(){ /* 從 Shell.store 取現值 */ },
  onNav: function(act){ return act==='VOL+'; }, // true=吃掉
  onVal: function(k,v){ /* 後端推來新 DB 值 */ }
});

v6 重點：層級用「名字」

Shell.LAYER_ORDER（唯一真源，後面＝高）

volume音量/靜音
info-banner切源 toast
osd-menuOSD / Source
informationDisplay 三段
setup-wizard開機精靈
factory工廠選單
no-signal無訊號
blackout遮黑
source.usb / hdmi / home來源底層

挑一個名字填 layer。

查自己在哪：
Shell.getLayerRank('osd-menu')

大者在上。多個畫面可共用同一層（互斥只顯示最上）。

為什麼名字 > 數字

舊版（絕對數字）

priority: 970
priority: 950
priority: 940
// 要插一層 / 調順序
// → 一堆檔的數字全要改

v6（具名有序表）

layer: 'osd-menu'
layer: 'volume'
// 要調順序
// → 搬動 LAYER_ORDER 一個名字
// 畫面檔零改動 ✓

smoke-test.js (e) 鎖住這個保證：搬一個名字 → 兩畫面上下對調，screen 程式碼不動。

按鍵 NAV / 資料 VAL

NAV：先給最上層 onNav(act)

  • 回 true = 我吃掉了
  • 回 false = shell 全域路由 ↓
MENU→ osd-menu
SOURCE→ source-menu
BACK→ 關最上層
VOL± / MUTE→ audio-volume
DISPLAY→ info-display

VAL：DB 值變動

  • shell 先更新 Shell.store[key]
  • 再呼叫每個畫面 onVal(key,value)
  • 你只挑關心的 key 反應

你不自己判斷硬體狀態（no_signal／音量／Information 欄位都由後端推）。你只畫畫面。

特殊：source-bound 底層

綁某來源、全屏、不由按鍵叫出（如 USB 播放器）

Shell.register({
  id: 'app-usb',
  layer: 'source.usb',
  source: 'USB',        // ← 關鍵：綁 USB 來源
  mount: function(el){ /* 畫播放器殼 */ },
  onNav: function(act){ return false; } // 讓系統鍵照常路由
});

切到 USB → shell 自動把它當不透明底層顯示（在所有 overlay 之下），切走自動隱藏。你不用 show/hide。

Do / Don't

✅ Do

  • 只碰自己的 el
  • 共享狀態讀 Shell.store
  • mount 建一次，刷新放 onShow
  • 改完跑 node smoke-test.js

❌ Don't

  • 伸手改別人畫面的 DOM
  • 自己寫絕對 z-index 跟別人比
  • 自己開 WebSocket / 判斷 no_signal
  • 改 shell.js / transport-ws.js
